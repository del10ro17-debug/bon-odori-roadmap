# 参加可否を「保存したら全員に見える」ようにする（坂倉さん・5分）

## みんなの使い方（設定後）

1. [公開ページ](https://del10ro17-debug.github.io/bon-odori-roadmap/bon-odori/) を開く
2. **参加・回答** タブ
3. 名前・時間帯を選んで **回答を保存**  
→ 全員の画面に反映（30秒〜1分）

JSONやトークンは**不要**です。

---

## 坂倉さんが最初に1回だけやること

### ① Google Apps Script を開く

https://script.google.com/ → **新しいプロジェクト**

### ② コードを貼る

1. リポジトリで **`open_bon_odori_sync_setup.command` をダブルクリック**  
   → コードがクリップボードに入り、Apps Script が開きます  
2. Apps Script の `コード.gs` で **`Cmd + A` → `Cmd + V` → `Cmd + S`**  
3. 左のファイル名が **「コード.gs」だけ** になり、**「まだ保存されていません」が消える**まで保存

※ 貼り付け後、行の先頭に変な空白が大量にある場合は貼り直してください。

### ③ setup（省略可）

初回アクセス時に自動でスプレッドシートを作るようにしてあります。  
手動でやる場合のみ:

1. 関数一覧で **`setup`** → **実行** → 権限を許可  
2. （任意）**`seedCurrentResponses`** で既存2件を投入

### ④ ウェブアプリとして公開

1. **デプロイ** → **新しいデプロイ**  
2. 種類: **ウェブアプリ**  
3. 実行ユーザー: **自分** / アクセス: **全員**  
4. **デプロイ** → 出た **URL**（`.../macros/s/.../exec`）をコピー

### ⑤ URL をページに設定

`docs/bon-odori/data.js` の `attendanceApiUrl` に貼る:

```javascript
attendanceApiUrl: "ここにコピーしたURL",
```

GitHub に push → 1〜2分後、全員が「保存だけ」で共有できます。

**デプロイでエラーが出たとき**  
→ ②の保存が終わっているか確認（`Cmd + S`）。保存後にもう一度 **デプロイ → 新しいデプロイ**。

---

設定が終わると、ページの「参加・回答」に **「自動共有: オン」** と表示されます。

**ウェブアプリ URL が取れたら** Cursor に貼ってください。`data.js` への反映と push までこちらで行います。
